🔧 服务器更新问题解决指南

⚠️ 问题描述

本地测试页面显示正常，但服务器上的更改没有生效。这通常是缓存问题导致的。

🚀 立即解决方案

步骤1: 强制刷新浏览器

Windows/Linux: Ctrl + F5 或 Ctrl + Shift + R

Mac: Cmd + Shift + R

或者: 打开开发者工具(F12) → 右键刷新按钮 → 选择"清空缓存并硬性重新加载"

步骤2: 清除浏览器缓存

Chrome: 设置 → 隐私设置和安全性 → 清除浏览数据 → 选择"缓存的图片和文件"

Firefox: 设置 → 隐私与安全 → Cookie和网站数据 → 清除数据

快捷方式: Ctrl + Shift + Delete

步骤3: 检查文件是否正确上传

确认以下文件已经上传到服务器:

  • templates/capital_flow.html
  • static/md3-styles.css

可以通过SSH或FTP工具检查文件的修改时间是否是最新的。

步骤4: 重启Web服务

如果使用的是Flask/Django等Python应用，需要重启服务:

  • Railway/Render: 重新部署应用
  • 本地服务器: 重启Python进程
  • Docker: 重启容器

🔍 高级排查方法

方法1: 添加版本号参数

在CSS和JS文件后添加版本号参数，强制浏览器重新加载:

static/md3-styles.css?v=20241201

方法2: 检查网络请求

打开浏览器开发者工具(F12) → Network标签 → 刷新页面 → 查看CSS/JS文件是否返回304(缓存)还是200(新内容)

方法3: 直接访问文件

直接在浏览器中访问CSS文件，检查内容是否已更新:

https://your-domain.com/static/md3-styles.css

✅ 验证修复是否成功

修复成功后，您应该看到:

  • 涨跌幅列只显示一个值(箭头+百分比)
  • 主力净流入列显示正确的资金数据
  • 操作列显示两个按钮
  • 所有列都正确对齐

🛡️ 预防措施

1. 添加缓存控制头

在Web服务器配置中为静态文件添加适当的缓存控制头。

2. 使用版本控制

为CSS和JS文件添加版本号或哈希值，确保更新时强制刷新。

3. 设置开发环境

在开发环境中禁用缓存，便于调试和测试。

💡 提示: 如果以上方法都不起作用，请检查是否在正确的服务器环境中，或者联系服务器管理员确认部署状态。
